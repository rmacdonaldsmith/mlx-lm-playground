Payment Form Validation Requirements

CONTEXT: E-commerce checkout form for US customers saving payment methods

FORM FIELDS (all mandatory):
- First Name: 2-50 characters, letters/spaces/hyphens only
- Last Name: 2-50 characters, letters/spaces/hyphens only  
- Card Number: 13-19 digits, must pass Luhn algorithm validation
- Address: 5-100 characters, alphanumeric with standard punctuation
- ZIP Code: US format only (5 digits or 5+4 format: 12345 or 12345-6789)

VALIDATION BEHAVIOR:
- Validate on field blur (when user tabs out)
- Show real-time validation for card number typing
- Block form submission if any validation fails
- Clear previous errors when user starts typing

ERROR MESSAGES:
- First/Last Name: "Name must be 2-50 characters using only letters, spaces, and hyphens"
- Card Number: "Please enter a valid credit card number" 
- Address: "Address must be 5-100 characters"
- ZIP Code: "Please enter a valid US ZIP code (12345 or 12345-6789)"
- Form submission: "Please correct the errors above before continuing"

SUPPORTED CARD TYPES: Visa, MasterCard, American Express, Discover
SECURITY: Never log or store raw card numbers in application logs
